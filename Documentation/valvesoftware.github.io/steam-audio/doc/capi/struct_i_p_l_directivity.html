Detailed Description

Specifies a directivity pattern.

A simple weighted dipole pattern may be specified. Alternatively, a callback may be specified to allow user-provided code to be called whenever the directivity pattern needs to be evaluated.

Field Documentation

◆ dipoleWeight

IPLfloat32 dipoleWeight

Controls the blend between a monopole (omnidirectional) and dipole directivity pattern.

0.0 means pure monopole, 1.0 means pure dipole. 0.5 results in a cardioid pattern.

◆ dipolePower

IPLfloat32 dipolePower

Controls the width of the dipole directivity pattern.

Higher values mean sharper, more focused dipoles.

◆ callback

IPLDirectivityCallback callback

Pointer to a function to call when the directivity pattern needs to be evaluated.

◆ userData
